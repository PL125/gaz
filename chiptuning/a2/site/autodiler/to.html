О программе АвтоДилер.

Версия 1.0
Система "АвтоДилер" представляет собой совокупность модулей (подсистем), каждый из которых предназначен для реализации конкретной задачи:

  • АвтоСтрахование (ОСАГО) - регистрация, заполнение и печать бланков заявлений ОСАГО, полисов ОСАГО.
  • ТехОсмотр - регистрация, заполнение и печать диагностических карт и технических талонов.

Модульная структура системы "АвтоДилер" позволяет поставлять конечному пользователю программную реализацию тех задач, которые ему интересны, на такие сроки, на которые ему необходимо их решение. В программе предусмотрена общая часть, представленная перечнем справочников, являющихся общими для всей системы, используемых в каждом из ее отдельных модулей. При поставке дополнительных модулей к программе конечного пользователя, они подключаются к уже наработанной пользователем базе, содержащей заполненные справочники по клиентам, валютам, сотрудникам, предприятиям, банкам, маркам и моделям автомобилей и т.д.
Политика лицензирования программного обеспечения предоставляет набор прав на использование каждого модуля системы "АвтоДилер" конечному пользователю в течение срока, указанного в лицензионном соглашении для каждого модуля. Такая реализация предоставляет пользователю возможность по необходимости подключать те модули системы, которые необходимы ему для решения текущих задач, на тот срок, на который требуется это решение.



Возможности системы "АвтоДилер"


Справочники.

  • Ведение справочника "Клиенты" с возможностью разбиения клиентской базы на группы и подгруппы в зависимости от функционального предназначения, котором наделяет пользователь регистрируемых в программе клиентов. Возможность регистрации по каждому клиенту набора свойств: контактов, идентификационных документов, реквизитов, автомобилей клиента. Каждое из этих свойств также представлено набором элементов (несколько контактов, документов, реквизитов, автомобилей). Сводный отчет по клиенту, включающий информацию по контактам, документам, реквизитам, автомобилям клиента, информацию по статусам клиента в других подсистемах, а также данные по заключенным на него (клиента) документам, зарегистрированным в других подсистемах.



  • Ведение справочника "Предприятия" с возможностью иерархического представления его данных, позволяющего визуально разграничить головные и подчиненные предприятия. Возможность регистрации по каждому предприятию набора свойств: контактов, идентификационных документов, реквизитов, сотрудников предприятия. Каждое из этих свойств также представлено набором элементов (несколько контактов, документов, реквизитов, сотрудников).



  • Ведение справочника "Сотрудники" с обязательной привязкой каждого сотрудника к предприятию и регистрации его свойств: контактов, идентификационных документов, реквизитов. Сводный отчет по сотруднику, включающий информацию по контактам, документам, реквизитам сотрудника, предприятиям, на которых он зарегистрирован, информацию по статусам сотрудника в других подсистемах.



  • Ведение справочника "Валюты" с возможностью регистрации в системе валют и курсов между ними на определенную дату, а также назначения валюты по умолчанию и национальной валюты.



  • Ведение справочника "Банки" с возможностью регистрации информации по банкам, используемых для оформления реквизитов (расчетных счетов, открытых в этих банках) клиентов, сотрудников, предприятий.



  • Ведение справочника "Марки и модели автомобилей" с возможностью разбиения моделей на группы и подгруппы в зависимости от функционального предназначения, которым наделяет пользователь регистрируемые в программе модели автомобилей.



  • Ведение справочников "Общие справочники", представляющих собой совокупность общесистемных информационных ресурсов ("Нормативные акты", "Подразделения", "Цвета", "Фирмы-производители" и т.д. ), участвующих в формировании документов и элементов других справочников.



  • Возможность настройки по каждому справочнику перечня отображаемых колонок, колонок, по которым выполняется сортировка данных. Если справочник представлен в виде нескольких зон, то существует возможность настройки размеров этих зон для каждого справочника системы.
  • В каждом справочнике системы существует возможность установки скрытых (неактуальных) элементов, а также элементов по умолчанию, что позволяет ускорить процесс заполнения документов, использующих данные справочников.
  • В каждом справочнике системы существует возможность установки режима отображения данных с настройкой "показывать/не показывать скрытые записи", что позволяет выполнять фильтрацию отображаемых данных (разграничить актуальные и неактуальные данные справочника).
  • Возможность выполнения локального поиска по таблице, позволяющего оперативно находить необходимые данные справочника.
  • Горячие клавиши, пункты контекстного меню, кнопки на панели инструментов дублируют доступ к основным функциям системы, позволяя пользователю выбрать наиболее удобный для себя вариант.
  • Возможность получения истории для любого элемента справочника, то есть, статистических данных, содержащих сведения о том, кем и когда элемент был создан, кем и когда он был изменен.


Сервисные функции.

  • Большое количество настраиваемых параметров системы.




  • Возможность работы с системой нескольких пользователей с разграничением прав доступа к различным частям программы.



  • Ведение истории, в которой отражены действия, производимые пользователями с основными объектами системы (кем и когда были созданы объекты, кем и когда они были изменены).
  • Предоставлено руководство пользователя, справочная система, контекстная справка, позволяющая оперативно открыть нужный раздел справочной системы.
  • Интуитивно понятный интерфейс.
  • Сетевая платформа программы, позволяющая работать с программой нескольким пользователям одновременно, используя общую базу.
  • Существует возможность настройки палитры отображения диалоговых окон в системе.
  • Возможность оперативного доступа к внешним программ из системы (калькулятор, календарь, проводник и т.д.).


Требования и рекомендации.

  • Программные средства
    Система "АвтоДилер" тестировалась на следующих операционных системах:
    • WindowsR 95, 98, ME, NT 4.0 (не поддерживается)
    • WindowsR 2000 Professional +последний пакет обновлений
    • WindowsR XP Professional or Home Edition +последний пакет обновлений (рекомендуется)
    • WindowsR Server 2003 Enterprise Edition +последний пакет обновлений (рекомендуется)
    • WindowsR Vista +последний пакет обновлений


    Для работы системы "АвтоДилер" требуется установка следующего программного обеспечения:
    • Firebird Server Win32 build, v.2 (входит в поставку)
    • Microsoft Internet Explorer 4 и выше (необходим для справочной системы)


  • Аппаратные средства
    • Процессор семейств IntelR (PentiumR / CeleronR / XeonT), AMD (AthlonT / DuronT) тактовая частота которых составляет не менее 800 МГц. Работа на процессорах AMD K5 и AMD K6 не поддерживается.
    • Минимальные требования к оперативной памяти зависят от используемой операционной системы (для Windows XP - не менее 256 Мб).
    • Видеоплата и монитор с разрешением не менее 1024х768 точек, глубина цвета не менее 16bit. Рекомендуемые параметры: разрешение 1280х1024 и более, глубина 16bit и более, мелкий шрифт.
    • Свободное место на жестком диске не менее 100 Mb.
    • Свободный USB-порт на сервере.
    • Клавиатура и манипулятор мышь.
    • Для сетевого варианта наличие локальной вычислительной сети. Установленный протокол TCP/IP.
    • CD-ROM привод (для установки системы на компьютере).

О программе АвтоДилер - ТехОсмотр.

  • Ведение справочника "Контролеры" с возможностью регистрации сотрудников из справочника "Сотрудники" в качестве контролеров, выполняющих функцию инструментального контроля в проведении техосмотра транспортного средства.



  • Ведение "Диагностических карт", являющихся документами, подтверждающими факт прохождения техосмотра, содержащих информацию по объекту техосмотра и по результатам проведения проверок диагностики и являющихся основание для выдачи клиенту техталона.



  • Возможность регистрации тарифов для групп моделей автомобилей, что позволяет вести стоимостной учет в разрезе диагностических карт, оформленных на каждую группу автомобилей.



  • Печать диагностической карты, содержащей результаты прохождения техосмотра и дополнительную информацию, идентифицирующую транспортное средство, его собственника и представителя, если он есть.
  • Печать технического талона, выдаваемого клиенту при прохождении техосмотра.
  • Сводный отчет по контролеру, включающий информацию по диагностическим картам, диагностика транспортного средства указанного в которых выполнена контролером, а также дополнительную информацию по сотруднику, которым является контролер.
  • Сводный отчет по техосмотру включает информацию по группам моделей автомобилей, а также количественный и стоимостной показатель в разрезе заключенных по каждой группе диагностических карт.
